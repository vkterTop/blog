# 博客视觉与布局设计规范

> 适用项目：Astro 静态博客（基于 `D:\my\guajie\sucai_questions` 文章数据）  
> 风格定位：高端大气 · 古典汉风 · 移动端优先 · 阅读为本  
> 版本：v1.0

---

## 1. 设计解读

**页面类型**：历史/文化类内容博客，以长文阅读为核心。  
**受众**：对汉服、疆域、军事、明清史、思想文化等议题有兴趣的中文读者。  
**设计语言**：「素笺墨韵」——以传统宣纸、朱砂、墨黑为基调，辅以现代栅格与留白，营造书斋案头的沉静阅读氛围。  
**设计系统倾向**：不依赖现成组件库，以 Tailwind CSS + 自定义 CSS 变量 + 原生字体栈实现。

### 三档设定

| 维度 | 取值 | 说明 |
|------|------|------|
| `DESIGN_VARIANCE` | 6 | 偏编辑式不对称，首屏与分类页略有错落，正文严格居中阅读栏。 |
| `MOTION_INTENSITY` | 3 | 以内容阅读为主，动态克制；仅保留滚动渐显、hover 微反馈。 |
| `VISUAL_DENSITY` | 3 | 大量留白，行距宽松，避免信息拥挤。 |

### 2.2 深色模式

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景主色 | `#121110` | 浓墨底色。 |
| 背景次色 | `#1a1816` | 卡片、表面层。 |
| 背景强调 | `#24211e` | hover、引用块。 |
| 文字主色 | `#e8e4dd` | 正文标题。 |
| 文字次色 | `#a8a39b` | 摘要、元信息。 |
| 弱化文字 | `#6f6a62` | 时间戳、占位符。 |
| 主强调色 | `#d85c5c` | 深色模式下朱砂提亮。 |
| 主强调悬停 | `#e87a7a` | 悬停态。 |

### 2.3 使用原则

- 以米白/墨黑为绝对主导，朱砂仅作点缀，不超过页面色彩面积的 8%。
- 古铜金仅用于装饰性元素（首屏副标题、引用引号、少量徽章），避免大面积使用。
- 所有交互元素必须有明确的 hover/focus 状态变化。
- 禁止使用纯黑 `#000000` 与纯白 `#ffffff`。

## 3. 字体系统

### 3.1 字体栈

```css
--font-display: "Noto Serif SC", "Source Han Serif SC", "STSong", "SimSun", serif;
--font-body: "Noto Serif SC", "Source Han Serif SC", "PingFang SC", "Microsoft YaHei", serif;
--font-mono: "JetBrains Mono", "Fira Code", "Noto Sans SC", monospace;
--font-decorative: "Ma Shan Zheng", "ZCOOL XiaoWei", "STKaiti", cursive; /* 仅用于印章或装饰标题 */
```

### 3.2 字体选择说明

- **Noto Serif SC / Source Han Serif SC**：思源宋体，笔画清晰，具有古籍排版的庄重感，同时屏幕可读性优秀。
- **Ma Shan Zheng**：一款书法体，仅用于小面积装饰（如首屏印章、分类徽章），禁止用于正文或长标题。
- **西文配对**：使用 `Crimson Pro` 或 `EB Garamond` 与中文宋体搭配，保持西文衬线气质。
- 由于中文字体体积较大，建议通过 Google Fonts 或国内 CDN 按需加载，并设置 `font-display: swap`。
- 如性能受限，可优先使用系统字体回退，正文仍以 `PingFang SC` / `Microsoft YaHei` 保证可读性。

### 3.3 字号与行距

| 元素 | 桌面端 | 平板端 | 移动端 | 行高 | 字间距 |
|------|--------|--------|--------|------|--------|
| 站点标题 | 2.25rem (36px) | 2rem | 1.75rem | 1.2 | 0.05em |
| 文章 H1 | 2.5rem (40px) | 2.25rem | 1.875rem | 1.35 | 0.02em |
| 文章 H2 | 1.875rem (30px) | 1.75rem | 1.5rem | 1.4 | 0.01em |
| 文章 H3 | 1.5rem (24px) | 1.375rem | 1.25rem | 1.45 | 0 |
| 正文 | 1.125rem (18px) | 1.0625rem | 1rem | 1.85 | 0.02em |
| 摘要/元信息 | 0.9375rem (15px) | 0.9375rem | 0.875rem | 1.7 | 0 |
| 小标签 | 0.8125rem (13px) | 0.8125rem | 0.75rem | 1.5 | 0.01em |

### 3.4 排版规则

- 正文段落最大宽度不超过 `42em`（约 34 个汉字/行），确保阅读舒适。
- 段落间使用 `margin-bottom: 1.75em` 而非空行分隔。
- 标题下方使用 1px 浅灰分隔线，或留足空白呼吸。
- 中文与西文/数字之间保留微小间距（通过 CSS `word-spacing` 或手动空格），但不超过 0.1em。
- 禁止对正文使用 `text-align: justify`，中文两端对齐易造成字间距不均。
- 深色模式下正文建议使用 `font-weight: 350` 或 400，避免纯白字过粗刺眼。

## 4. 布局系统

### 4.1 栅格与容器

- 最大内容宽度：`max-width: 1280px`。
- 正文阅读宽度：`max-width: 720px`。
- 页面水平边距：移动端 `16px`，平板 `24px`，桌面 `32px`。
- 采用 12 列栅格，但首页与分类页使用非对称组合打破呆板。

### 4.2 页面结构

```
BaseLayout
├── 顶部导航（固定，半透明宣纸色，带底部分隔线）
├── 主内容区
│   ├── 首页：首屏 + 精选文章 + 分类入口 + 最新文章列表
│   ├── 分类页：分类标题 + 文章卡片网格 + 分页
│   ├── 文章详情：文章头图区 + 正文 + 关键词 + 上下篇导航
│   ├── 归档页：时间线列表
│   └── 搜索页：搜索框 + 结果列表
└── 页脚（简洁版权与返回顶部）
```

### 4.3 首页布局

#### 首屏（Hero）

- 高度：`min-h-[70dvh]`，避免全屏阻塞滚动。
- 左侧：站点名称（书法体印章形式） + 一句话描述 + 分类快速入口。
- 右侧：抽象水墨纹理背景或装饰性竖排文字（如「经史子集」四字）。
- 背景：米白宣纸色 +  subtle 噪点纹理 + 底部水墨渐变。
- 移动端：改为垂直堆叠，右侧装饰文字缩小或隐藏。

#### 分类入口

- 使用 2 列网格（桌面），1 列（移动端）。
- 每个分类卡片为长方形，带分类名称、文章数量、一句代表性关键词。
- 卡片 hover 时微微上浮（`translateY(-4px)`），阴影加深，左侧出现朱砂色竖线。

#### 最新文章列表

- 桌面端：左侧时间线 + 右侧文章列表（非对称 1/3 + 2/3）。
- 移动端：单栏时间线卡片。
- 每条文章包含：标题、分类、摘要（2 行）、日期、关键词标签。

### 4.4 文章详情页布局

- 顶部文章头：标题、分类、日期、关键词。
- 头图区（可选）：若文章无封面，则使用抽象水墨占位图或纯色纹理。
- 正文容器：最大 720px，居中，两侧留白充足。
- 右侧桌面端可加入：目录导航（TOC）sticky 悬浮，但需保证不干扰阅读。
- 文章底部：关键词、上下篇导航、返回分类按钮。
- 移动端：目录收起为顶部按钮或隐藏。

### 4.5 分类页布局

- 顶部大标题 + 分类描述（2-3 句）。
- 文章卡片网格：桌面 3 列，平板 2 列，移动端 1 列。
- 分页：简洁数字分页，当前页使用朱砂色背景圆点。

### 4.6 响应式断点

| 断点 | 宽度 | 主要行为 |
|------|------|----------|
| `sm` | 640px | 移动端基础样式，单栏。 |
| `md` | 768px | 平板，部分网格变为 2 列，导航展开。 |
| `lg` | 1024px | 桌面，完整栅格，侧边 TOC 出现。 |
| `xl` | 1280px | 大屏，最大容器居中，增大间距。 |

## 5. 组件规范

### 5.1 导航栏（Navbar）

- 高度：桌面 64px，移动端 56px。
- 背景：`#f7f4ef` / 深色 `#121110`，带 `backdrop-blur(8px)` 与底部 1px 分隔线。
- 左侧：站点名称（宋体，字重 600）。
- 中间：分类入口（桌面可见，移动端收起）。
- 右侧：搜索图标 + 深色模式切换图标。
- 移动端：汉堡菜单展开全屏遮罩，菜单项纵向排列，字大、间距宽。

### 5.2 文章卡片（PostCard）

- 背景：比页面底色略深的 `#efe9e0`。
- 无圆角或仅 2px 圆角（保持古典方正感）。
- 内边距：24px。
- 结构：分类 eyebrow（小字） → 标题 → 摘要 → 底部（日期 + 关键词）。
- hover：左侧出现 4px 朱砂竖线，整体 `translateY(-4px)`，阴影加深。
- 无图片时默认显示水墨占位纹理。

### 5.3 按钮

- 主按钮：朱砂背景 + 米白文字，圆角 2px，hover 亮度降低。
- 次按钮：透明背景 + 朱砂边框 + 朱砂文字，hover 背景填充。
- 文字链接：朱砂色，下划线仅在 hover 出现，transition 0.2s。
- 所有按钮：active 时 `scale(0.98)`，提供触感反馈。

### 5.4 标签与徽章

- 关键词标签：小字、细边框、圆角 2px，hover 背景变为淡朱砂。
- 分类徽章：朱砂填充 + 米白文字，仅用于分类页标题旁。
- 印章式徽章：仅在首屏或特殊区域使用，采用书法体 + 朱砂色圆形/方形背景。

### 5.5 分页（Pagination）

- 水平排列，数字按钮间距 8px。
- 当前页：朱砂背景圆点（或方点），米白文字。
- 非当前页：透明背景，文字色。
- 上一页/下一页：箭头图标 + 文字，hover 时箭头微移。

### 5.6 搜索组件

- 搜索框：宽输入框，细边框，背景同色，focus 时边框变朱砂。
- 搜索按钮：朱砂背景，米白文字。
- 搜索结果：标题 + 分类 + 摘要高亮，hover 背景变为 `#efe9e0`。

### 5.7 引用块

- 左侧 4px 古铜金竖线。
- 背景：`#efe9e0`（浅色）或 `#1a1816`（深色）。
- 字号比正文小 1px，行距 1.7。
- 引号使用古铜金大号引号装饰（不使用 emoji）。

### 5.8 页脚

- 背景：与页面底色相同或略深。
- 上边框 1px 分隔线。
- 内容：站点名 + 版权声明 + 链接列表（首页、归档、搜索、RSS）。
- 返回顶部按钮：右下角固定，圆形，朱砂背景，hover 上浮。

## 6. 装饰与纹理

### 6.1 允许使用的装饰元素

- **宣纸噪点**：使用固定伪元素，低透明度 PNG 或 SVG 噪点，覆盖整个页面，不随滚动重绘。
- **水墨渐变**：仅用于首屏底部过渡、文章头图占位。
- **竖排文字**：仅用于首屏装饰或分类页大标题局部，不得用于正文。
- **分隔线**：1px  hairline，颜色为 `#d8d2c9`（浅色）或 `#2f2c28`（深色）。
- **印章图标**：小面积 SVG 印章，用于首屏或分类徽章。

### 6.2 禁止使用的装饰元素

- 霓虹光晕、紫色渐变、蓝色科技光效。
- 圆角过大的卡片（超过 8px）。
- 漂浮的几何装饰动画。
- 自定义鼠标指针。
- 大面积 emoji。

## 7. 动效与交互

### 7.1 动画原则

- 动效必须为阅读服务，而非炫技。
- 所有动画仅使用 `transform` 与 `opacity`，避免触发布局重排。
- 尊重 `prefers-reduced-motion`，在媒体查询中关闭非必要动画。

### 7.2 具体动效

| 场景 | 效果 | 参数 |
|------|------|------|
| 页面加载 | 首屏内容淡入 + 上移 | `opacity: 0→1`, `translateY(20px→0)`, 0.6s, ease-out |
| 文章列表进入视口 | 卡片依次淡入 | `opacity: 0→1`, `translateY(16px→0)`, 0.5s, stagger 0.08s |
| 卡片 hover | 上浮 + 阴影 | `translateY(-4px)`, `box-shadow` 加深，0.25s ease |
| 按钮 hover | 背景色变化 + 轻微缩放 | `scale(1.02)`, 0.15s |
| 按钮 active | 按下反馈 | `scale(0.98)` |
| 链接 hover | 下划线出现 | `border-bottom` 或 `text-decoration`，0.2s |
| 返回顶部 | 平滑滚动 | `scroll-behavior: smooth` |
| 移动端菜单 | 全屏淡入 | `opacity` + 内容滑入，0.3s |

### 7.3 禁用行为

- 禁止滚动劫持、视差滚动、水平滚动画廊。
- 禁止无限循环的 micro-interaction（如脉冲、闪烁）。
- 禁止使用 `window.addEventListener('scroll')` 直接监听滚动。

---

## 8. 深色模式实现

- 使用 Tailwind `dark:` 变体或 CSS 变量切换。
- 默认跟随系统 `prefers-color-scheme`。
- 提供手动切换按钮，状态存储于 `localStorage`。
- 切换时无过渡动画，避免页面闪烁。

## 9. Astro / Tailwind 实现建议

### 9.1 CSS 变量定义

在 `src/styles/global.css` 中定义：

```css
:root {
  --paper: #f7f4ef;
  --paper-2: #efe9e0;
  --paper-3: #e7e0d5;
  --ink: #1a1714;
  --ink-2: #5e5851;
  --ink-3: #9b948b;
  --cinnabar: #b83a3a;
  --cinnabar-hover: #9c2e2e;
  --bronze: #8c6b3d;
  --hairline: #d8d2c9;
}

.dark {
  --paper: #121110;
  --paper-2: #1a1816;
  --paper-3: #24211e;
  --ink: #e8e4dd;
  --ink-2: #a8a39b;
  --ink-3: #6f6a62;
  --cinnabar: #d85c5c;
  --cinnabar-hover: #e87a7a;
  --bronze: #c4a66f;
  --hairline: #2f2c28;
}
```

### 9.2 Tailwind 配置扩展

```js
// tailwind.config.mjs
theme: {
  extend: {
    colors: {
      paper: 'var(--paper)',
      'paper-2': 'var(--paper-2)',
      'paper-3': 'var(--paper-3)',
      ink: 'var(--ink)',
      'ink-2': 'var(--ink-2)',
      'ink-3': 'var(--ink-3)',
      cinnabar: 'var(--cinnabar)',
      'cinnabar-hover': 'var(--cinnabar-hover)',
      bronze: 'var(--bronze)',
      hairline: 'var(--hairline)',
    },
    fontFamily: {
      serif: ['"Noto Serif SC"', '"Source Han Serif SC"', 'PingFang SC', 'Microsoft YaHei', 'serif'],
      display: ['"Ma Shan Zheng"', '"ZCOOL XiaoWei"', 'STKaiti', 'cursive'],
      mono: ['"JetBrains Mono"', '"Fira Code"', 'monospace'],
    },
    maxWidth: {
      'prose-chinese': '42em',
    },
    lineHeight: {
      'chinese': '1.85',
    }
  }
}
```

### 9.3 字体加载策略

- 在 `BaseLayout.astro` 中通过 `<link>` 引入 Google Fonts：
  - `Noto Serif SC`（正文）
  - `Ma Shan Zheng`（装饰）
  - `Crimson Pro`（西文）
- 设置 `font-display: swap`。
- 若追求性能，可仅在首屏使用 `Ma Shan Zheng`，正文使用系统宋体/黑体回退。

### 9.4 移动端优先类名约定

- 默认样式为移动端，逐步向 `md:`、`lg:` 扩展。
- 导航栏在 `md` 以下使用汉堡菜单。
- 文章列表在 `md` 以下为单栏卡片。
- 正文阅读容器在所有屏幕下保持 `max-w-prose-chinese`。

---

## 10. 可访问性

- 所有文字对比度满足 WCAG AA 标准，正文目标 AAA。
- 按钮与链接 focus 态必须有清晰可见的朱砂色 outline。
- 图片必须提供 `alt` 文本。
- 导航菜单支持键盘操作与屏幕阅读器。
- 尊重 `prefers-reduced-motion`，禁用非必要动画。
